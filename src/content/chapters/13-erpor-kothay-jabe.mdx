পৌঁছে গেলে। অভিনন্দন। 🍵 ১৩টা quest শেষ।

তুমি এখন TCA-র ভেতরের পাঁচ বন্ধুকে চেনো — state, action, reducer, store, effect। তুমি জানো — কখন এই pattern reach করতে হবে, কখন না। তুমি একটা ছোট real project লিখেছ। তুমি test লিখেছ।

## যা শিখোনি — কিন্তু পরে দরকার পড়বে

এই site-এ basics covered। আরো কিছু advanced জিনিস আছে যেগুলো প্রয়োজন হলে শিখবে —

### `Shared` state

App-এর একাধিক feature যদি same state share করে — singleton-জাতীয় কিছু (logged-in user, app settings) — তখন `@Shared` property wrapper use হয়। Docs-এ "SharedState" article দেখো।

### Long-running effects

WebSocket connection, location updates, file watchers — `.run` দিয়ে শুরু, কিন্তু cancel-able. `.cancellable(id:)`, `.merge(...)`, `Effect.publisher(...)` — এই tools।

### `Reducer` builders গভীরে

`Scope`, `IfLet`, `ForEach`, `OptionalReducer` — সব combine করে complex feature বানানো। কোনো এক জায়গায় bottleneck লাগলে — docs-এর "Reducer composition" article।

### Animation আর state

State change-এর সাথে animation — `withAnimation { store.send(...) }`-এর equivalent TCA-তে আছে। SwiftUI animation conventions এর সাথে আগে আগে চলে।

### Migration থেকে পুরনো TCA

পুরনো codebase-এ পড়লে — `Reducer<S, A, E>`, `.pullback`, `WithViewStore` — সেগুলো আজ modernize করতে হলে — repo-র Migration guide-এ ধাপে ধাপে instructions আছে।

## কিছু অভ্যাস যা সাহায্য করবে

- **প্রতিটা feature আলাদা ফাইলে** — `XxxFeature.swift`, `XxxView.swift`। বড় file ১৫০-২০০ লাইনের বেশি হতে দিও না।
- **নাম রাখো consistent**: action-এ `xxxTapped`, `xxxResponse`, `xxxFailed`। 
- **State pure**: state-এ কোনো method, কোনো computed property যেটা reducer-এর কাজ করে, রাখো না। শুধু derived data ok।
- **প্রতিটা bug-fix-এ একটা test**: feature change করার সাথে সাথে test add — debt জমতে দিও না।
- **Junior dev-কে শেখাও**: যা শিখলে, সেটা অন্যকে শেখাতে গিয়ে নিজে আরও পরিষ্কার বুঝবে।
